# Type System

GoyGram uses Python `__slots__` for zero-overhead field storage in all core event objects. This eliminates `__dict__`, saving memory during high-throughput event processing.

## Core Event Objects

### MsgObj

Represents an incoming message from any transport.

```python
class MsgObj:
    __slots__ = ("src", "raw", "app", "id", "chat_id", "from_id",
                 "text", "is_me", "cmd", "args", "match", "finds", "parts")
```

| Field | Type | Description |
|-------|------|-------------|
| `src` | `str` | `"bot"` or `"mt"` |
| `raw` | `dict` | Original normalized event dict |
| `app` | `AppCore` | Runtime reference |
| `id` / `msg_id` | `int\|None` | Message ID |
| `chat_id` | `int\|str\|None` | Chat/peer ID |
| `from_id` | `int\|None` | Sender user ID |
| `text` | `str` | Message text (always a string) |
| `is_me` | `bool` | From current account? |
| `cmd` | `str\|None` | Matched command name (set by `command` filter) |
| `args` | `str\|None` | Command arguments (set by `command` filter) |
| `match` | `re.Match\|None` | Regex match object (set by `regex`, `fullmatch`) |
| `finds` | `list\|None` | Regex find results (set by `findall`, `finditer`) |
| `parts` | `list[str]\|None` | Split results (set by `split` filter) |

`cmd`, `args`, `match`, `finds`, and `parts` are set by filter classes via `object.__setattr__` during filter evaluation — they carry filter results into handler code.

Methods: `reply()`, `delete()`, `net()`, `_resolve_peer()`.

### CbObj

Callback query events from inline keyboard buttons.

```python
class CbObj:
    __slots__ = ("src", "raw", "app", "id", "chat_id", "from_id",
                 "msg_id", "data", "text", "match", "payload", "json_data")
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | `str\|None` | Callback query ID (from `query_id` or `id`) |
| `chat_id` | `int\|str\|None` | Source chat ID |
| `from_id` | `int\|None` | User who pressed the button |
| `msg_id` | `int\|None` | Message ID with the keyboard |
| `data` | `str` | `callback_data` from the button |
| `text` | `str` | Original message text or caption |
| `match` | `re.Match\|None` | Regex match (set by `cb_regex` filter) |
| `payload` | `tuple\|None` | Parsed payload (set by `cb_payload` filter) |
| `json_data` | `Any\|None` | Parsed JSON (set by `cb_json` filter) |

Methods: `answer()`, `edit()`.

### PollObj

Poll update events.

```python
class PollObj:
    __slots__ = ("src", "raw", "app", "id", "question", "closed", "kind")
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | `str\|int\|None` | Poll ID |
| `question` | `str` | Poll question text |
| `closed` | `bool` | Whether the poll is closed |
| `kind` | `str` | Always `"poll"` |

### MemberObj

Chat member status change events.

```python
class MemberObj:
    __slots__ = ("src", "raw", "app", "chat_id", "from_id",
                 "user_id", "old", "new", "kind")
```

| Field | Type | Description |
|-------|------|-------------|
| `chat_id` | `int\|None` | Chat where the change occurred |
| `from_id` | `int\|None` | User who initiated the change |
| `user_id` | `int\|None` | Affected user ID |
| `old` | `str\|None` | Previous status |
| `new` | `str\|None` | New status |
| `kind` | `str` | Always `"member"` |

Status values: `"creator"`, `"administrator"`, `"member"`, `"restricted"`, `"left"`, `"kicked"`.

## Keyboard Builder

All keyboards are built through a single dynamic `KbdBuilder` class with no hardcoded button fields. Accessed via `app` factory methods:

```python
# Inline keyboard — any button field passes through as-is
kbd = app.ikb().btn("Click", callback_data="act").btn("URL", url="https://...").build()

# Reply keyboard with options
kbd = app.rkb(resize_keyboard=True).btn("Option").build()

# Force reply / remove keyboard
kbd = app.frk(selective=True).build()
kbd = app.rgk(selective=True).build()
```

`.btn(text, **kw)` passes `**kw` directly into the button dict — no field is hardcoded. `.row()` starts a new button row. `.build()` (aliased as `.to_dict()`) returns the final dict.

For link preview options, pass a plain dict:

```python
await msg.reply("text", link_options={"is_disabled": True})
```

Full reference: [Keyboard System](Keyboard-System).

## Bot API Types

Generated by `tools/gen_botapi.py` in `goygram/api/types.py`. All use `__slots__` and `to_dict()`:

- `User` — `id`, `is_bot`, `first_name`, `username`
- `Chat` — `id`, `type`, `title`, `username`
- `Message` — `message_id`, `date`, `chat`, `text`

Keyboard types use `KbdBuilder` instead of hardcoded classes.

## The `dump()` Helper

Universal recursive serialization used before JSON encoding:

```python
def dump(v: Any) -> Any:
    if hasattr(v, "to_dict"):
        return v.to_dict()
    if isinstance(v, list):
        return [dump(x) for x in v]
    if isinstance(v, dict):
        return {k: dump(x) for k, x in v.items() if x is not None}
    return v
```

Dictionaries with `None` values have those keys stripped to avoid sending `null` to the API.

## TL Schema Types

MTProto types in `goygram/tl/schema.py` extend `TlObj`:

```python
class TlObj:
    __slots__ = ()
    cid = 0
    res = ""
    def to_dict(self) -> dict: ...
    def to_bytes(self) -> bytes: ...
```

Hand-written TL objects: `ResPQ`, `PQInnerData`, `Ping`, `MsgsAck`, `InvokeWithLayer`. Each has a hardcoded `cid` and manual `to_bytes()` serialization. The `REG` dict maps constructor IDs to classes for deserialization.

For dynamic TL serialization at runtime, the Rust extension's `serialize_method()` handles any method defined in the loaded `.tl` schema — no Python-side type definitions needed for standard MTProto calls.
